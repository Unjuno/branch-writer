# Concept

## What Branch Writer Is

Branch Writer は ChatGPT wrapper ではありません。通常のチャットUIが「プロンプトを送信し、完成した返答を読む」という非同期の往復モデルであるのに対し、Branch Writer は「生成過程に割り込み、任意位置から再開する」同期的 co-writing モデルを採用しています。

## What Makes It Different

### ユーザーは操縦者である

一般的なチャットでは、ユーザーは「入力してから待つ」受動的立場です。Branch Writer では、ユーザーは生成中のストリームに対して常に割り込む権利を持ちます。AI が書き進める文章に対し、人間が任意の時点で「ここからは自分が書く／ここを直す」と介入できます。

### snapshot を正とする

介入が発生した時点で、その瞬間のテキスト全体のsnapshotを取得し、それを新しいstart地点とします。編集中にstreamが進んでもsnapshotは変わらず、ユーザーがEnterを押した時点の内容が再生成の起点になります。

### suffix は常に破棄される

介入時の選択位置より後ろのテキストは常に破棄されます。AIが直前に生成していた内容も、人間の介入判定によって無意味とみなされれば捨てられます。この動作は仕様であり、バグではありません。

## Design Decisions

### 過去メッセージ編集は意図的にしない

過去のAssistantメッセージを編集可能にすると、以下の問題が生じるため採用していません。

- コンテキスト全体の再計算が必要になる
- どの時点の介入がどのコンテキストで行われたか追跡不能になる
- UI の複雑性が増す

### branch tree UI は意図的にしない

v0 では単一の直線的な編集履歴を採用します。複数候補の比較・統合は将来の課題です。

### データベースは使わない

設定も会話履歴もファイルに保存しません。セッションが切れれば失われます。これはプロトタイプフェーズでは複雑性より再現性を優先する判断です。

### contenteditable は使わない

ブラウザの contenteditable はcursor位置の正確な取得が難しく、streaming更新との整合性が取れないため、textarea を採用しています。

## Goal

AI に「答えを出させる」のではなく、人間がAIの生成を操縦しながら「文章を仕上げる」ことを可能にします。目標は「AI案の採択」ではなく「人間主導の継続生成」です。
