# Known Limitations and Non-goals

## Current Limitations

| 制限 | 理由 |
|---|---|
| 介入できるのは最新Assistantメッセージのみ | 過去メッセージ編集はコンテキスト再計算と追跡不能性の問題がある |
| 過去メッセージは編集不可（凍結） | 同上 |
| 分岐ツリーUIはない | v0では単一直線の編集履歴。複数候補の比較は将来課題 |
| 自動矛盾検出はない | 介入によって生じた前後矛盾を自動検出・修正する仕組みは未実装 |
| 永続化はない | 設定・会話履歴ともに保存しない。セッション終了で失われる |
| カーソルループ機能は削除済み | 旧cursor_loopモードはReact textarea移行に伴い削除。復活予定なし |

## Explicit Non-goals（意図的にやらないこと）

| 非目標 | 理由 |
|---|---|
| フルcontenteditableエディタ | cursor位置の正確な取得が困難。textareaで十分 |
| 過去Assistant編集 | コンテキスト整合性が取れない |
| 複数候補UI (A/B比較) | v0のスコープ外。直線編集に集中 |
| branch tree UI | v0のスコープ外 |
| データベース層 | プロトタイプでは複雑性より再現性を優先 |
| カーソルループ復活 | 現行のinline介入方式で代替可能 |
