# Usage

## 1. 会話を始める

1. Ollama もしくは LM Studio などのローカル LLM サーバーを起動する
2. `streamlit run app.py` を実行する
3. サイドバーでモデルを選択する（空欄のAPI URLで自動検出されます）
4. メッセージを入力して送信する

## 2. AI の返答を確認する

AI の返答はリアルタイムでストリーミング表示されます。生成中でも以下の操作が可能です。

## 3. 介入して再生成する

1. 最新のAssistantメッセージのtextarea内で、再生成を開始したい位置にテキストカーソルを移動する
2. 必要に応じて介入文を入力する（空のままでも可）
3. Enter を押す
4. カーソル位置以降のテキストが破棄され、そこからの続きが再生成される

### 介入文の例

| 操作 | 内容 |
|---|---|
| 空Enter | その位置から続きを再生成 |
| 「具体的に」と入力してEnter | 「具体的に」という指示を前置して再生成 |
| 文章を修正してEnter | 修正後の文章をprefixとして続きを生成 |

## 4. 生成中の割り込み

生成中でも同じ操作で割り込めます。現在のストリームは中断され、新しいストリームがカーソル位置から開始されます。

## 5. 編集のキャンセル

Escape キーで編集を中断し、textareaの内容を最新のストリーム内容に戻せます。ストリーム自体は止まりません。

## 6. 直前の介入を取り消す

サイドバーの「元に戻す」ボタンで、直前の介入操作を取り消せます。

## 注意事項

- 介入できるのは最新のAssistantメッセージのみです
- 過去のメッセージは編集できません（凍結）
- ストリーム生成中にtextareaにフォーカスすると、新しいトークンの表示は一時停止しますが、バックグラウンドでは蓄積され続けます
- Shift+Enter は改行として機能し、再生成はトリガーしません
- IME（日本語入力）の変換中はEnterが再生成をトリガーしません
